# Telegram, MAX, Multy и Baserow

## Адрес приложения

Для Telegram и MAX используется одна опубликованная версия:

`https://neyroclay.github.io/mirofaktura-app/`

Приложение само определяет мессенджер и передаёт в колоду ID и имя пользователя.

## Таблица Baserow

Используется таблица `893270`. Текущих полей достаточно:

- `user_id` — ID пользователя;
- `messenger` — `TELEGRAM` или `MAX`;
- `first_name`, `last_name`;
- `first_launch_time`, `last_date`;
- `collected_cards`;
- `bonus_cards`, `invited_friends`.

Отдельный `profile_key` не нужен. Во всех запросах запись ищется по двум полям одновременно: `user_id` + `messenger`. Так ID из Telegram и MAX не попадут в одну строку.

## Первый экран колоды

- Если Multy не нашёл запись, приложение показывает вступительный экран.
- После нажатия кнопки колода сохраняет первую запись.
- При следующем открытии Multy возвращает `exists: true`, и колода открывается сразу.

## Сценарии Multy

Готовые файлы лежат в `_private/multy-ready/` и не попадают в GitHub. В `multy-schemas/` лежат те же схемы с заменённым токеном.

1. `01-shared-load-progress.json` — загрузка прогресса для обоих мессенджеров.
2. `02-shared-save-progress.json` — сохранение прогресса для обоих мессенджеров.
3. `03-max-referral.json` — текущая реферальная логика MAX с фильтром по `messenger`.

После импорта Multy выдаст новые адреса двух вебхуков. Их нужно передать в приложение перед публикацией.

## Приветствие Telegram

В Telegram не нужен отдельный шаг с политикой и согласием. Сценарий `/start` отправляет одно сообщение:

> Добро пожаловать в Мирофактуру.
>
> Мы — Елизавета Викулова и Елена Попова. Соединяем системный маркетинг, геймификацию и новые способы вовлечения.
>
> В приложении можно пройти короткую диагностику, получить подарок под текущую задачу, открыть интерактивные инструменты и вытянуть карту из колоды трендов 2026.

Кнопка: `Открыть Мирофактуру`.

## Приветствие MAX

В MAX можно сохранить текущий шаг согласия. После него нужно отправлять то же приветствие и кнопку на тот же URL.

## Безопасность

Токен Baserow должен оставаться только в Multy. Нельзя добавлять его в HTML, JavaScript, GitHub или публичную документацию.
