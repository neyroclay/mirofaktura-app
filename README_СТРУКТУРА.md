# Мирофактура: структура приложения

Текущая сборка оставлена простой для локального просмотра и GitHub Pages:

- `index.html` — вход для GitHub Pages.
- `preview.html` — локальный предпросмотр.
- `styles.css` — все визуальные правила.
- `app.js` — данные, рендер страниц и клики.
- `assets/` — изображения, логотип и персонажи.

## Почему пока не дробим сильнее

Приложение должно открываться и через `file://`, и с GitHub Pages без сборщика. Поэтому сейчас лучше не переходить на импорт модулей: локально в браузере они могут вести себя иначе, чем опубликованная версия.

Оптимальный шаг на сейчас — держать код в отдельных файлах `HTML / CSS / JS`, а внутри `app.js` добавлять новые страницы через общий реестр.

## Как добавлять страницу

1. Добавить данные рядом с `quiz` и `materials`, если странице нужны карточки или тексты.
2. Создать функцию `renderName()`.
3. Добавить страницу в `PAGE_RENDERERS`.
4. Если странице нужен нижний пункт меню, добавить ее в `PAGES_WITH_BOTTOM_NAV` и кнопку в `bottomNav()`.
5. Добавить обработчик `data-action` или `data-page` в общий click listener.
6. Стили держать отдельным блоком в `styles.css`. Общие кнопки, меню и карточки менять только после проверки главной, квиза и кладовой.

## Перед заметной правкой

- сохранить копии `app.js` и `styles.css` в `_checkpoints/`;
- проверить `node --check app.js`;
- открыть `preview.html` и пройти главную, квиз, результат и кладовую.
