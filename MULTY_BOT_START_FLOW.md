# Новый старт из бота Multy

Основа: файл `graph-29690-1783581325267.json` можно использовать как источник, но не стоит импортировать его в текущем виде. Он сделан под старую колоду трендов: там много сообщений по дням, выдача отдельных карт и старые тексты.

Новый смысл сценария проще: человек открывает бота, соглашается с документами, подписывается на нужный канал или каналы, потом получает кнопку входа в приложение.

## Что оставить из старого графа

В старом графе уже есть нужная часть про согласие:

- условие: CRM-поле `0b9e0c0f3de3286` равно `да`;
- сообщение с текстом согласия и кнопкой `Продолжить`;
- действие, которое после нажатия пишет в поле `0b9e0c0f3de3286` значение `да`.

Эту часть можно оставить. Она должна идти до проверки подписки и до выдачи кнопки приложения.

## Что убрать из старого графа

Из нового старта не нужно переносить:

- тексты про "первый цифровой артефакт";
- кнопку `Открыть портал`;
- таймеры, которые выдают вторую, третью, четвертую карту;
- старые продажи полной колоды;
- тексты "карманная вселенная", "стратегический навигатор", "живая система" и похожие формулировки.

Это было под отдельную механику колоды. Сейчас приложение шире: квиз, кладовая, материалы, тренды и контакты.

## Новый порядок сценария

### 1. Старт

Если согласие уже есть, сразу переходить к проверке подписки.

Если согласия нет, показать сообщение:

```html
Перед входом в приложение нужно подтвердить согласие с документами.

Продолжая, вы соглашаетесь с <a href="ССЫЛКА_НА_ПОЛИТИКУ">Политикой обработки персональных данных</a>, даете <a href="ССЫЛКА_НА_СОГЛАСИЕ">согласие на обработку персональных данных</a> и соглашаетесь получать сообщения от проекта.
```

Кнопка:

```text
Продолжить
```

После нажатия записать в CRM-поле согласия:

```text
да
```

### 2. Проверка подписки

Для MAX:

- проверять подписку на канал Мирофактуры в MAX;
- если подписки нет, показать кнопку подписки и кнопку повторной проверки.

Для Telegram:

- проверять подписку на два Telegram-канала авторов;
- доступ выдавать только если подписка есть на оба канала;
- если одной подписки не хватает, показать понятный текст и кнопки на оба канала.

Текст для неподписанного пользователя:

```html
Материалы в приложении открываются после подписки.

Подпишитесь на канал, вернитесь сюда и нажмите "Проверить подписку".
```

Кнопки:

```text
Подписаться
Проверить подписку
```

### 3. Выдача входа в приложение

Если согласие есть и подписка найдена, отправить короткое сообщение:

```html
Готово, доступ открыт.

Откройте приложение: там можно пройти диагностику, забрать материалы из кладовой и открыть колоду трендов.
```

Кнопка:

```text
Открыть приложение
```

Для MAX это должна быть app-кнопка на актуальную страницу мини-приложения.

Для Telegram это должна быть кнопка открытия Telegram Mini App.

## Что нужно будет заменить после деплоя

В старом графе app-кнопки ведут на старую страницу:

```text
startapp=trends
```

После деплоя и настройки мини-приложения в MAX нужно заменить все старые app-кнопки на актуальный вход в новое приложение.

Важно: кнопка должна открывать не отдельную колоду, а главный экран приложения.

## Минимальный безопасный вариант

Если хочется не трогать сложный старый граф, лучше сделать новый короткий сценарий в Multy:

1. `/start`.
2. Проверить поле согласия.
3. Если согласия нет, показать согласие и кнопку `Продолжить`.
4. Записать согласие.
5. Проверить подписку.
6. Если подписки нет, показать кнопки подписки и повторной проверки.
7. Если подписка есть, показать кнопку `Открыть приложение`.

Так будет легче поддерживать две версии: MAX и Telegram.

## Что важно для безопасности

- Не выдавать кнопку приложения до согласия и проверки подписки.
- Не хранить экспорт этого графа в публичном GitHub, если внутри есть интеграции, CRM-поля, токены или служебные id.
- Для MAX и Telegram лучше сделать две копии сценария: так меньше риск перепутать каналы, платформу и записи в Baserow.
- В тексте бота не писать "MAX", если этот же сценарий потом будет копироваться для Telegram. Лучше писать нейтрально: "вернитесь сюда" или "откройте приложение".
